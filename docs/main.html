3D Muscle Segmentation – Results Notebook¶

Authors: Sabina Grüner, Michelle Rohrer · FHNW · FS 2026

Automatic voxel-wise segmentation of 8 right thigh muscles (Quadriceps: VL, RF, VM, VI; Hamstrings: BfSH, BfLH, ST, SM) from water-weighted 3D MRI. Method: Bayesian 3D U-Net with Monte Carlo Dropout, 3-Fold Cross-Validation.

This notebook presents the full solution process: Overfit Test → Baseline → Hyperparameter Experiments → Evaluation → Analysis.

For the exploratory data analysis see notebooks/EDA.ipynb.

1 · Overfit Test¶

Before running cross-validation, we verify that the model has sufficient capacity by overfitting on a single subject (512). Target: Dice ≥ 0.90. This follows the Karpathy recipe: prove the model can memorize before asking it to generalize.

def load_scalars(logdir: Path, tag: str) -> tuple[list[int], list[float]]:
    """Read one scalar tag from a TensorBoard log directory."""
    ea = EventAccumulator(str(logdir))
    ea.Reload()
    events = ea.Scalars(tag)
    steps = [e.step for e in events]
    vals  = [e.value for e in events]
    return steps, vals

def load_fold_curves(exp_key: str, tag: str = 'val/macro_dice') -> list[tuple[list, list]]:
    """Load val dice curves for all three folds of an experiment."""
    curves = []
    for fold in ['fold0', 'fold1', 'fold2']:
        logdir = RUNS / exp_key / fold
        if logdir.exists():
            steps, vals = load_scalars(logdir, tag)
            curves.append((steps, vals))
    return curves


def plot_curves(ax, curves, color, label):
    max_len = min(len(c[1]) for c in curves)
    arr = np.array([c[1][:max_len] for c in curves])
    steps = curves[0][0][:max_len]
    mean = arr.mean(axis=0)
    lo, hi = arr.min(axis=0), arr.max(axis=0)
    ax.plot(steps, mean, color=color, linewidth=1.8, label=label)
    ax.fill_between(steps, lo, hi, color=color, alpha=0.15)

7 · MC Dropout Uncertainty¶

Expected Calibration Error (ECE) measures how well the model's confidence (from N=20 MC Dropout samples) correlates with actual accuracy. Lower = better calibrated.

ece_rows = []
for key, label in EXPERIMENTS:
    eval_path = CKPT / key / 'eval_report.json'
    d = json.loads(eval_path.read_text())

8 · Summary¶

RQ1 (Dice ≥ 0.60): Not achieved. Best result: hpt_combined 0.308 (0.355 with CC). The overfit Dice of 0.90 proves model capacity; the generalization gap stems from n=10 training subjects. Clinical-grade segmentation typically requires 30–50 subjects.

RQ2 (improvement ≥ 0.03 vs. baseline): Achieved. hpt_combined improves by +0.074 (without CC) and +0.089 (with CC). The combination of FocalTversky loss (penalises false negatives, β=0.7 > α=0.3) and class-balanced sampling (uniform draw over classes 1–8) is the key driver.

Three error modes identified from confusion matrix progression:

  • Baseline: Hamstring collapse into background (5–16% correct)
  • hpt_focal: Background collapse eliminated, but inter-muscle confusion (BfSH 55%, VM drops to 51%)
  • hpt_combined: Quadriceps reliable (VL 92%, VI 94%), Hamstrings systematically weak (BfSH 0%)

Failed experiments: hpt_augmentation (Dice 0.126) due to an anatomical augmentation bug — Axis-0/1 flips swap VL↔VM and RF↔Hamstrings, producing contradictory training signal. hpt_dropout (0.130) confirms that p=0.5 is too aggressive for n=10.
